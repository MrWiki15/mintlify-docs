Kawi permite que empresas y desarrolladores integren capacidades de pagos, conversión y liquidación directamente dentro de sus productos.

Con la API de Kawi, una empresa puede:

- Crear depósitos.
- Consultar balances.
- Ejecutar conversiones.
- Enviar fondos.
- Recibir webhooks.
- Integrar un widget embebible.
- Usar Kawi como infraestructura de remesas o liquidación.

## Autenticación

Todas las peticiones a la API de Kawi requieren autenticación mediante API key.

### Header básico

```http
Authorization: Bearer KAWI_API_KEY
Content-Type: application/json
```

### Ejemplo

```bash
curl https://api.kawi.life/v1/balance \
  -H "Authorization: Bearer KAWI_API_KEY"
```

### Buenas prácticas

- No expongas API keys en frontend.
- Usa variables de entorno.
- Rota claves periódicamente.
- Separa claves de sandbox y producción.

## Referencia de API

Esta referencia inicial muestra los endpoints base previstos para integraciones.

### Obtener balance

```http
GET /v1/balance
```

Respuesta de ejemplo:

```json
{
  "available": "1520.00",
  "currency": "BRL",
  "updated_at": "2026-06-23T12:00:00Z"
}
```

### Crear depósito

```http
POST /v1/deposits
```

Body:

```json
{
  "amount": 100,
  "currency": "BRL",
  "method": "PIX",
  "customer_id": "cus_123"
}
```

Respuesta:

```json
{
  "id": "dep_123",
  "status": "pending",
  "amount": 100,
  "currency": "BRL",
  "payment_method": "PIX"
}
```

### Crear conversión

```http
POST /v1/conversions
```

Body:

```json
{
  "from_currency": "BRL",
  "to_currency": "USDC",
  "amount": 1000
}
```

Respuesta:

```json
{
  "id": "conv_123",
  "status": "processing",
  "from_currency": "BRL",
  "to_currency": "USDC",
  "estimated_amount": "185.40"
}
```

### Crear transferencia

```http
POST /v1/transfers
```

Body:

```json
{
  "amount": 50,
  "currency": "USDC",
  "destination": "wallet_address_or_user_id"
}
```

Respuesta:

```json
{
  "id": "trf_123",
  "status": "sent",
  "amount": 50,
  "currency": "USDC"
}
```

## Webhooks

Los webhooks permiten recibir notificaciones automáticas cuando ocurre un evento.

### Eventos disponibles

```text
deposit.created
deposit.completed
conversion.started
conversion.completed
transfer.sent
transfer.failed
withdrawal.completed
```

### Ejemplo de payload

```json
{
  "event": "conversion.completed",
  "data": {
    "id": "conv_123",
    "from_currency": "BRL",
    "to_currency": "USDC",
    "status": "completed"
  }
}
```

### Buenas prácticas

- Valida la firma del webhook.
- Responde con HTTP 200.
- Procesa eventos de forma idempotente.
- Guarda logs de eventos recibidos.

## Widget de Kawi

El widget de Kawi permite integrar flujos de conversión y remesas en una plataforma externa con pocas líneas de código.

### Ejemplo HTML

```html
<script
  src="https://cdn.kawi.life/widget.js"
  data-app-id="YOUR_APP_ID"
  data-theme="light"
  data-height="720">
</script>
```

### Qué permite el widget

- Mostrar servicios de Kawi dentro de otra web.
- Procesar conversiones fiat ↔ cripto.
- Crear flujos de remesas.
- Permitir onboarding sin construir una interfaz desde cero.

### Casos de uso

- Fintechs.
- Marketplaces.
- Apps de comunidad.
- Plataformas de remesas.
- Negocios con usuarios internacionales.

## Laboratorio para desarrolladores

El laboratorio para desarrolladores es el entorno de pruebas de Kawi.

Permite:

- Simular depósitos.
- Probar conversiones.
- Validar webhooks.
- Testear integraciones.
- Revisar errores.
- Ejecutar flujos sin dinero real.

Su objetivo es que cualquier desarrollador pueda validar una integración antes de pasar a producción.

## Portal para desarrolladores

El portal para desarrolladores es el panel donde las empresas gestionan su integración.

Incluye:

- API keys.
- Logs.
- Webhooks.
- Estadísticas.
- Estado de operaciones.
- Configuración del widget.
- Ambientes sandbox y producción.
